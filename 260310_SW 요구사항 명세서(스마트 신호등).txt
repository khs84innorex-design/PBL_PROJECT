# 소프트웨어 요구사항 명세서 (SRS)
## 프로젝트명: 스마트 신호등 프로젝트

---

## 1. 문서 개요

### 1.1 목적
본 문서는 LED와 버튼을 이용한 스마트 신호등 프로젝트의 소프트웨어 요구사항을 정의한다.  
요구사항은 구현 가능하고 테스트 가능하도록 기능 요구사항과 비기능 요구사항으로 구분하여 기술한다.

### 1.2 범위
본 시스템은 단일 방향 차량 흐름을 제어하는 소형 신호등 시스템이다.  
차량 신호는 초록, 노랑, 빨강 LED로 표현되며, 사용자의 버튼 입력을 통해 보행 요청을 처리한다.

### 1.3 대상 독자
- 개발자

### 1.4 용어 정의
- **차량 신호**: 차량의 진행/주의/정지를 나타내는 LED 신호
- **보행 요청**: 사용자가 버튼을 눌러 보행 신호를 요청하는 행위
- **상태(State)**: 시스템이 특정 시점에 가지는 동작 모드

---

## 2. 시스템 개요

### 2.1 시스템 설명
스마트 신호등 시스템은 차량용 신호등의 기본 동작을 LED로 구현하며, 사용자가 보행자 버튼을 누르면 현재 차량 흐름을 안전하게 마무리한 후 보행 가능한 상태를 제공한다.

### 2.2 시스템 목표
- 차량 신호등의 기본 순차 제어를 구현한다.
- 버튼 입력을 통해 보행 요청을 등록하고 처리한다.
- 잘못된 신호 조합 없이 안전한 순서로 상태를 전환한다.
- 현재 상태를 사용자가 쉽게 식별할 수 있도록 한다.

### 2.3 가정
- 본 프로젝트는 **단일 교차 흐름**만 고려한다.
- 차량 신호용 LED는 **빨강 1개, 노랑 1개, 초록 1개**를 사용한다.
- 보행자 신호는 **추가 LED 또는 차량 빨강 상태를 통해 간접 표현**할 수 있다.

## 7. 상태 전이 규칙

| 규칙 ID | 설명 |
|---|---|
| SR-001 | 시스템 시작 시 ST-001(차량 초록)으로 진입해야 한다. |

| SR-007 | 보행 요청 처리 완료 후 시스템은 ST-001로 복귀해야 한다. |
| SR-008 | 상태 전이 중 충돌 LED 조합이 발생해서는 안 된다. |

## 8. 인터페이스 요구사항

### 8.1 사용자 인터페이스
- 사용자는 버튼 1개를 통해 보행 요청을 입력할 수 있어야 한다.
- 사용자는 LED 점등 상태를 통해 현재 시스템 상태를 인지할 수 있어야 한다.

### 8.2 하드웨어 인터페이스
- LED는 디지털 출력 핀에 연결되어야 한다.
- 버튼은 디지털 입력 핀에 연결되어야 한다.

## 12. 확인 필요 사항

아래 항목은 요구사항의 세부 수치를 확정하기 위해 사용자 확인이 필요하다.

1. **신호 유지 시간**
   - 초록, 노랑, 빨강 상태의 정확한 유지 시간은 몇 초로 할지 확인이 필요하다.

2. **보행 가능 상태 표현 방식**
   - 보행자용 별도 LED를 사용할지, 차량 빨강 상태만으로 보행 가능을 간주할지 확인이 필요하다.

3. **보행 처리 시간**
   - 보행 요청 처리 시 보행 가능 상태를 몇 초 유지할지 확인이 필요하다.

4. **구현 방식**
   - `delay()` 기반 단순 구현인지, `millis()` 기반 비차단 구현인지 결정이 필요하다.

## 13. 결론
본 명세서는 스마트 신호등 프로젝트의 소프트웨어 요구사항을 기능 및 비기능 요구사항으로 구분하여 정의하였다.
각 요구사항은 고유 ID, 우선순위, 검증 방법을 포함하며, 구현 가능성과 테스트 가능성을 고려하여 작성되었다.
